Responsive Web Design(RWD)

Es una metodologia de trabajo que busca que tanto el layout como el contenido de una pagina o aplicacion se vea correctamente en cualquier tama;o de viewport

No existen propiedades de CSS especificas para responsive Design por lo tanto,RWD es un conjunto de practicas :

  • Mobile firts (comenzar dise;ando la version movil e ir subiendo de tamaño
  • Content firts(tener el cuenta el contenido para aplicar cambios segun el tamaño de pantalla

Meta viewport

Es un tipo de meta introducido por Apple que le dice al navegador como interpretar las dimensiones de la pagina respecto a las dimensiones del viewport

El meta viewport no tiene ningun efecto en navegadores de escritorio

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Media Queries

Son reglas (at-rules) que permiten condicionar los estilos siempre que se cumplan ciertas condiciones.Inicialmente se usan para identificar los tipos de medios (impresora,print;pantalla,screen)pero tambien pueden identificar tamaño del viewport,orientacion del dispositivo o densidad de pixeles. 

    
        @media screen {
            body{
                background: #000;
                color: #fff;
            }
        }
        @media print{
            body{
                background: #fff;
                color:#333;
            }
        }
        @media all{
            body{
                font-family: Arial, Helvetica, sans-serif;
            }
        }

Las condiciones se unen con el operador and

    
        @media print and (min-width:200px){
            body{
                color: red;
            }
        }

Las condiciones pueden usar las comas como el operador logico or

    
        @media print,(min-width:200px){
            body{
                color: red;
            }
        }

Tambien se puede detectar la orientacion del dispositivo:

    
        @media screen and (orientation:portrait){
            //dispositivo en vertical
        }
        @media screen and (orientation:landscape){
            //dispositivo en horizontal
        }

Breakpoints

Son medidas relativas al ancho del viewport en las que se cambia el diseño Existen dos tipos:

  • Major Breakpoints(para cambios mayores como layout)
  • Minor Breakpoints(para cambios pequeños) se definen sobre el desarrollo del proyecto

Los breakpoints se combinan con las media queries

Es mas usual construir las media queries usando los breakpoints como min widht e ir subiendo de tamaño

Sass para RWD

Las media queries se pueden anidar al selector


        body{
            background-color: red;
            @media screen and (min-width:400px){
                background-color: green;
            }
            @media screen and (min-width:700px){
                background-color: yellow;
            }
            @media screen and (min-width:1000px){
                background-color: purple;
            }
        }

otra forma en la que Sass ayuda al RWD es que se pueden crear mixins para simplificar las media queries 

    
        @mixin desde($size){
            @media screen and (min-width:$size){
                @content;
            }
            @mixin hasta($size){
                @media screen and (max-width:$size){
                    @content;
                }
                }
            }
            @mixin desde-hasta($desde,$hasta){
                @media screen and (min-width:$desde) and (max-width:$hasta){
                    @content;
                }
                }

            header{
                background: red;
                @include desde(400px){
                    background: yellow;
                }
                @include desde(800px){
                    background: green;
                }
            }

Resolucion vs viewport

La Resolucion la cantidad de pixeles fisicos en una pantalla(ancho x alto) mientras que el viewport es el tamaño real que existe para diseñar. Generalmente ambos conceptor coincidian. pero cambia a partir del iphone 4 y el retina Display
