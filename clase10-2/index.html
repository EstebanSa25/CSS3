Preparar el entorno para trabajar con Gulp

  1. Instalar NodeJS
  2. Instalar NodeJS (nodejs.org)
  3. instalar git(opcional, pero recomendado en windows)
  4. Revisar que Node se instalo correctamente(comando "node -v")
  5. Inicializar el proyecto escribiendo en terminal "npm init"(este comando crea el package.json) *Solo se requiere si es que se esta creando un proyecto desde cero y no tiene el package.json en caso contrario se usa "npm install"
  6. Instalar Gulp
    • 1.Globalmente (solo la primera vez) "npm install gulp -g"
    • 2.Comprobar que esta instalado Gulp "gulp -v"
    • 3.Instalarlo localmente en el proyecto "npm install gulp --save-dev"
  7. Crear el gulpfile.js en la raiz del proyecto.Este archivo contiene las instrucciones para que gulp ejecute las tareas

Sass

Es un preprocesador css que permite escribir codigo css aprovechando las funcionalidad de un lenguaje de programacion como variables,funciones,scope,condicionales,etc.

Variables

Permiten almacenar valores en un contenedor y luego llamarlos cuando sea necesario,se escriben con un signo de dolar al inicio y su valor se asigna con dos puntos (:) 

        
    $size:20px;
    div{
        width:$size; //width:20px;
        height:$size;//height:20px;
    }

Anidamiento (nesting)

Se permiten poner selectores dentro de otros para representar unn selector decendiente 


    .padre{
        font-size: 2em;
        .hijo{
            font: {
                size:.8em;
                weight:normal;
                family:sans-serif;
                style:italic;
            }
            border: {
                width: 2px;
                color: green red yellow;
                radius:10px
            }
        }
    }

tambien se puede referenciar al selector padre usando el signo "&"


    .menu{
        list-style: none;
        &__item{
            display: inline-block;
        }
        &__link{
            display: block;
        }
        footer &{
            background: #000;
        }
        &.secundario{
            font-size: .8em;
        }
    }

Sass y Scss

Los archivos con las extension .sass usan una sintaxis antigua que no requeria punto y coma ni llaves Mientras que los archivos .scss usan la sintaxis actual (la anterior .sass ya no se desarrolla) y son los recomendados

para transformar un archivo .css a Sass basta con cambiarle la extension a .scss

Partials e @import

Los partials en Sass son los archivos que no deben compilarse por que se importaran dentro de otro. Por convencion esto se nombran comenzando con un guion bajo

Por lo general se acostumbra que un archivo sea el que importe a los demas con @import La sintaxis es la siguiente(no se necesita escribir el guion bajo ni la extension)


@import "componentes/menu;" //_menu.scss
@import "componentes/login" //_login.scss

Mixins

Son bloques de codigo que pueden recibir parametros

@extend y placeholder

@extend es para heredar propiedades de otra regla


    .menu{
        @include reset-menu(green,.9em)
    }

    .sociales{
        display: inline-block;
        line-height: 3rem;
        padding: 0 2em;
        &--footer{
        @extend .sociales;
    background-color: red;
        }

Un placeholder es un bloque de codigo reutilizable A diferencia de un mixin,un placeholder no recibe parametros. 


    %button{
        display: inline-block;
        line-height: 2;
        padding: 0 1em;
    }
    .button{
        @extend %button;
    }
